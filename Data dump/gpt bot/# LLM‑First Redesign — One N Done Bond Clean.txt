# LLM‑First Redesign — One N Done Bond Clean

**Objective:** Make every page trivially consumable by LLM crawlers *and* persuasive for humans.

You’re on Astro + Tailwind + Netlify. Below is a complete, drop‑in redesign blueprint: IA, copy skeletons, internal linking rules, schema, and code scaffolding. Build it incrementally; every section is production‑ready.

---

## 0) Core rules (LLM + SEO)

* **One page = one intent.** Each URL answers a single high‑intent question.
* **Stable anchors.** All sections get `id=` anchors using kebab case; never change once published.
* **Dates everywhere.** Stamp numerics (last updated, counts) with ISO dates.
* **Machine‑readable summary.** Each page ships a compact JSON digest `<script type="application/json+llm">` with facts LLMs can ingest.
* **Evidence inline.** Claims → short proof (photo, stat with date, policy snapshot).
* **Canonical + breadcrumbs.** Always present; no alias links.
* **Topical links only.** Each page links: up to hub, laterally to 2–4 peers, down to 2–6 children.
* **No ghost strings.** Error texts render only when true (no hidden “Submission failed.” in the DOM).

* **Service hub →** links to: 1) pricing, 2) checklist, 3) 12 suburb spokes, 4) 3 proof blogs.

## 4) JSON‑LD strategy (complete)

Embed all as separate `<script type="application/ld+json">` blocks:

1. **WebSite** (with SearchAction)
2. **Organization** or **LocalBusiness** (with `areaServed` list and `knowsAbout` services)
3. **Service** (Bond Cleaning) with `offers` (fixed price description), `areaServed`, `provider`
4. **FAQPage** on pages that contain FAQs
5. **BreadcrumbList** on all pages

> Do **not** invent ratings. Only publish `AggregateRating` if you can substantiate with a live reviews page.

### `/src/components/Schema.astro` (server‑only)

```astro
---
import type { AstroGlobal } from 'astro';
const site = {
  name: 'One N Done Bond Clean',
  url: Astro.site?.href ?? 'https://ondlive3.netlify.app/',
  phone: '+61-XXX-XXX-XXX',
  areas: ['Ipswich','Brisbane','Logan'],
};
const website = {
  '@context': 'https://schema.org',
  '@type': 'WebSite',
  name: site.name,
  url: site.url,
  potentialAction: {
    '@type': 'SearchAction',
    target: `${site.url}search?q={query}`,
    'query-input': 'required name=query'
  }
};
const org = {
  '@context':'https://schema.org',
  '@type':'LocalBusiness',
  name: site.name,
  url: site.url,
  telephone: site.phone,
  areaServed: site.areas.map(a=>({ '@type':'City', name:a })),
  knowsAbout: ['bond cleaning','end of lease cleaning','exit clean']
};
const service = {
  '@context':'https://schema.org',
  '@type':'Service',
  serviceType:'Bond cleaning',
  provider: { '@type':'LocalBusiness', name: site.name, url: site.url },
  areaServed: site.areas,
  offers: {
    '@type':'Offer',
    priceSpecification: { '@type':'PriceSpecification', priceCurrency:'AUD', price:'0', description:'Fixed-price quotes; price varies by property size and condition.' }
  }
};
---
<script type="application/ld+json">{JSON.stringify(website)}</script>
<script type="application/ld+json">{JSON.stringify(org)}</script>
<script type="application/ld+json">{JSON.stringify(service)}</script>
```

## 18) Why this works for LLM crawlers

* **Digest block** gives models a clean, non‑marketing snapshot.
* **Anchored sections** let extractive models jump straight to inclusions/pricing/coverage.
* **Dense internal links** raise discovery of spokes without orphaning.
* **Schema triad** (WebSite, LocalBusiness, Service) aligns with common toolchains.
* **Dates on claims** reduce hallucination risk and help time‑aware models.
